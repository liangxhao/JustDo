# HEARTBEAT.md 模板

`HEARTBEAT.md` 位于智能体工作区中。若希望 OpenClaw 跳过心跳模型调用，请保持此文件为空，或只保留 Markdown 注释和标题。

默认运行时模板可以保持为：

```markdown
# 保持此文件为空（或仅包含注释），即可跳过心跳 API 调用。

# 当你希望智能体定期检查某些事项时，请在下方添加任务。
```

只有在你确实希望智能体定期检查某些事项时，才在注释下方添加简短任务。心跳指令会在周期性唤醒时读取，因此请保持精简。

## 适合写进 HEARTBEAT.md 的内容

- 检查是否有新的高优先级事项
- 回顾中断任务的下一步
- 定期整理每日记忆
- 做轻量级、低风险的后台检查

## 不适合写进 HEARTBEAT.md 的内容

- 大规模或长时间运行的任务
- 精确到分钟的调度需求
- 需要明确用户确认的外部动作
- 依赖复杂上下文的大型执行流程

## 建议

- 让任务短、清楚、可判断是否完成
- 避免写入会反复触发噪声通知的内容
- 如果需要记录检查状态，可使用如 `memory/heartbeat-state.json` 之类的轻量文件
